Unofficial managed-agent sample

Gemini Anything Agent

A sample showing how one managed agent can become a persistent container for generating many kinds of content. The app gives users one chat surface; the managed agent handles reasoning and normal work; the gai CLI is used only for specialized media capabilities.

Architecture

One chat surface, one managed agent, specialized media on demand.

The agent is the center of gravity. It decides when normal managed-agent tools are enough and when a specialized model capability should be routed through the CLI.

Components

The sample has three moving parts.

1. Electron App

The app in app/ is the local chat harness. It stores conversations locally, ensures the managed agent exists, streams agent activity, keeps continuity, downloads generated media, and renders previews.

  • Uses previous_interaction_id for conversation context.
  • Reuses environment_id for remote workspace files.
  • Saves media locally under outputs/managed-agent/.

2. Managed Agent

The agent definition in agents/ mounts prompts, skills, the CLI wrapper, and environment configuration into a remote Linux sandbox.

  • Workspace root: /workspace
  • Durable outputs: /workspace/output
  • Mounted instructions: /.agents

3. gai CLI

The CLI in cli/ is published as @lyalindotcom/gai. It wraps specialized GenAI media calls behind a small command surface.

  • gai image
  • gai video
  • gai tts
  • gai transcribe

Runtime Flow

What happens when a user asks for something?

User sends a prompt.The app targets one configured agent, not a model picker.
The app ensures deployment.If the saved prompt, skill, wrapper, or environment sources changed, the app recreates the managed agent.
Continuity is attached.The app passes prior interaction and environment pointers by default, so the agent can remember context and reuse files.
The agent classifies the job.Code, tasks, analysis, research, browsing, and file edits use native managed-agent tools.
Specialized media routes through gai.New image, video, speech, and audio transcription tasks call the mounted CLI wrapper.
Artifacts come back to the app.Files are written to /workspace/output, downloaded from the remote environment, saved locally, and previewed inline.

Scenarios

The important behavior is generic routing, not one-off fixes.

Normal answer

Summarize the project and suggest the next engineering tasks.

The managed agent answers directly with native reasoning and file tools.

Image generation

Make me a square app icon for a CLI that generates anything.

The agent uses gai image, writes an output file, and the app previews it.

Podcast generation

Look at Hacker News and create a short recap podcast.

The agent researches, writes a script, then uses gai tts for the audio.

Existing artifact transformation

Convert that WAV podcast to MP3.

The agent inspects /workspace/output and uses normal shell tools like ffmpeg; it should not call gai for conversion.

Transcription

Transcribe the latest podcast with timestamps and speaker labels.

The agent resolves the source audio and calls gai transcribe.

Video generation

Make a short video of a cute cat playing with string in a cozy home.

The agent uses gai video. Video can take longer and cost more, so guardrails matter.

Setup

Run the sample locally.

App

cd app
npm install
npm run dev

Live API calls require Electron because the API key stays in the main process.

CLI

cd cli
npm install
npm test
npm run build
npm run gai -- models

The live package is @lyalindotcom/gai.

Environment

cp .env.example .env

GEMINI_API_KEY=
GEMINI_ANYTHING_AGENT_ID=gemini-anything-agent
GEMINI_ANYTHING_NPM_PACKAGE=@lyalindotcom/gai
GEMINI_ANYTHING_NPM_VERSION=latest
GEMINI_ANYTHING_TRANSCRIBE_MODEL=gemini-3.5-flash

Key handling

The app reads the local repo-root .env, then mounts a generated plaintext .env file into the managed-agent remote sandbox. The wrapper at /.agents/bin/gai sources that file before running the npm package, so media commands can call live APIs.

This is proof-of-concept wiring, not encrypted secret management. Use restricted or throwaway keys, quotas, billing alerts, and rotation. Do not commit real keys or treat snapshots and logs as secret-safe.

Safety

Do not treat this as production-ready.

This sample intentionally gives an agent broad ability to reason, browse, run code, create files, and call specialized generation APIs. That is the point of the experiment, and it is also why cost and control guardrails are required before real use.
