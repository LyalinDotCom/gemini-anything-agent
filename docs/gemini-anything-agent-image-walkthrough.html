The idea

Gemini Anything Agent uses one chat surface and one managed agent. The agent does normal work directly: text, code, research, analysis, browsing, and file inspection. When a request needs a specialized media model, the agent calls a tiny CLI wrapper instead of guessing API details.

Electron app

Owns the local UX: conversations, previews, output files, local cache, and agent deployment.

Managed agent

Runs remotely, keeps continuity, decides what to do, and writes artifacts into the workspace.

gai CLI

Wraps the GenAI SDK for image, video, TTS, and transcription through one simple command surface.

The main pattern: the agent is the general-purpose worker, while gai is the narrow media tool. That keeps model selection and media API details out of the app UI.

The create-image flow

When a user asks the app to create an image, the app does not call an image model directly. It starts or resumes a conversation with the managed agent and lets the agent choose the right action.

The app checks configuration.It reads the local .env and verifies that a Gemini API key is available.
The app creates or refreshes the agent.On first chat start, the app deploys the agent definition, prompt files, skill file, wrapper script, and generated sandbox .env.
The user message goes to the managed agent.Conversation continuity and environment continuity are sent when available.
The agent decides this is media work.Its instructions tell it to use native tools for normal work and /.agents/bin/gai for new media generation.
The CLI creates the image.The command writes the generated file to /workspace/output.
The app downloads and caches outputs.Images, videos, audio, text, HTML, and other files appear in the local output panel without repeating downloads that are already cached.

Wrapping the GenAI SDK

The CLI wrapper is intentionally small. It selects a model, calls the GenAI SDK, decodes the returned image data, and writes the file. The published package is @lyalindotcom/gai on npm.

image-wrapper.ts
import { GoogleGenAI } from "@google/genai";
import { writeFile } from "node:fs/promises";

const ai = new GoogleGenAI({
  apiKey: process.env.GEMINI_API_KEY
});

export async function generateImage(prompt: string, out = "image.jpg") {
  const interaction = await ai.interactions.create({
    model: process.env.GEMINI_IMAGE_MODEL ?? "gemini-3.1-flash-image",
    input: prompt,
    response_format: {
      type: "image",
      mime_type: "image/jpeg",
      aspect_ratio: "1:1",
      image_size: "1K"
    }
  } as never);

  const image = (interaction as any).output_image;
  if (!image?.data) {
    throw new Error("The image model did not return output_image.data");
  }

  await writeFile(out, Buffer.from(image.data, "base64"));
  return out;
}

Inside the managed-agent sandbox, the wrapper script uses npx so the agent can run the latest published package without bundling the CLI into the agent source.

/.agents/bin/gai
#!/usr/bin/env bash
set -euo pipefail
if [ -f /.env ]; then set -a; . /.env; set +a; fi
export NODE_USE_ENV_PROXY="${NODE_USE_ENV_PROXY:-1}"
exec npx -y "${GEMINI_ANYTHING_NPM_PACKAGE:-@lyalindotcom/gai}@${GEMINI_ANYTHING_NPM_VERSION:-latest}" "$@"

Creating the managed agent

The app creates a Gemini Managed Agent with code execution, Google Search, URL context, a remote environment, and inline files that become available inside the sandbox. The key files are the system prompt, the skill instructions, the gai wrapper, and a generated .env.

This copyable TypeScript sketch mirrors the shape used by the app. Replace the system prompt and skill text with your own instructions.

Agent runtime

  • Remote Linux sandbox
  • /workspace as the working directory
  • /workspace/output as the artifact handoff folder
  • /.agents/bin/gai as the media command
create-agent.ts
const API_BASE = "https://generativelanguage.googleapis.com/v1beta";
const API_REVISION = "2026-05-20";
const AGENT_ID = process.env.GEMINI_ANYTHING_AGENT_ID ?? "gemini-anything-agent";
const GAI_PACKAGE = process.env.GEMINI_ANYTHING_NPM_PACKAGE ?? "@lyalindotcom/gai";
const GAI_VERSION = process.env.GEMINI_ANYTHING_NPM_VERSION ?? "latest";

const apiKey = process.env.GEMINI_API_KEY;
if (!apiKey) {
  throw new Error("Set GEMINI_API_KEY before creating the agent.");
}

const gaiWrapper = `#!/usr/bin/env bash
set -euo pipefail
if [ -f /.env ]; then set -a; . /.env; set +a; fi
export NODE_USE_ENV_PROXY="\${NODE_USE_ENV_PROXY:-1}"
exec npx -y "\${GEMINI_ANYTHING_NPM_PACKAGE:-${GAI_PACKAGE}}@\${GEMINI_ANYTHING_NPM_VERSION:-${GAI_VERSION}}" "$@"
`;

const systemInstruction = `
You are Gemini Anything Agent.
Use native managed-agent tools for text, code, research, analysis, and file work.
Use /.agents/bin/gai only for new image, video, text-to-speech, and transcription work.
When creating files, save them under /workspace/output and report the path.
If the user refers to an existing artifact, inspect /workspace/output first.
`;

const skillText = `
# Gemini Anything Skill

Use /.agents/bin/gai --help to discover the available media commands.
For image generation, run:

/.agents/bin/gai image "<prompt>" --out /workspace/output/image.jpg --json

Keep user-facing prompts clean. Put technical output-path instructions in the skill or system prompt.
`;

const agent = {
  id: AGENT_ID,
  base_agent: "antigravity-preview-05-2026",
  system_instruction: systemInstruction,
  tools: [
    { type: "code_execution" },
    { type: "google_search" },
    { type: "url_context" }
  ],
  base_environment: {
    type: "remote",
    sources: [
      { type: "inline", target: ".agents/bin/gai", content: gaiWrapper },
      { type: "inline", target: ".agents/skills/gemini-anything/SKILL.md", content: skillText },
      {
        type: "inline",
        target: ".env",
        content: [
          `GEMINI_API_KEY=${apiKey}`,
          `GEMINI_ANYTHING_NPM_PACKAGE=${GAI_PACKAGE}`,
          `GEMINI_ANYTHING_NPM_VERSION=${GAI_VERSION}`,
          ""
        ].join("\n")
      }
    ]
  }
};

const response = await fetch(`${API_BASE}/agents`, {
  method: "POST",
  headers: {
    "Content-Type": "application/json",
    "x-goog-api-key": apiKey,
    "x-goog-api-revision": API_REVISION
  },
  body: JSON.stringify(agent)
});

if (!response.ok) {
  throw new Error(await response.text());
}

console.log(await response.json());

Sending the request

Once the agent exists, each chat turn is an interaction. The app sends the user input plus continuity pointers. Conversation continuity keeps the agent aware of prior turns. Environment continuity keeps the sandbox and files available across turns.

interaction-request.ts
const request = {
  agent: "gemini-anything-agent",
  input: "Create a square image of a tiny robot assistant brewing coffee.",
  previous_interaction_id: previousInteractionId,
  environment: previousEnvironmentId ?? { type: "remote" },
  background: true,
  store: true
};

const response = await fetch(`${API_BASE}/interactions`, {
  method: "POST",
  headers: {
    "Content-Type": "application/json",
    "x-goog-api-key": process.env.GEMINI_API_KEY!,
    "x-goog-api-revision": API_REVISION
  },
  body: JSON.stringify(request)
});

The result is not just a text answer. The agent can leave files in /workspace/output, and the local app turns that folder into a browsable output panel with previews and downloads.

Boundaries

This is an unofficial proof of concept. It uses your Gemini API key, can trigger real API spend, and mounts plaintext .env content into the managed-agent sandbox for simplicity.

That tradeoff is useful for a sample because it makes the architecture easy to inspect: the app deploys the agent, the agent calls gai, gai calls Gemini media models, and the app previews the files. For anything beyond a prototype, replace the sample secret flow with your own production-grade controls.
